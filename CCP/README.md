# 《计算机网络》综合课程设计

## 任务要求
1. 选择 1 个题目完成
2. 提交源程序和报告文档

## 选题列表

1. TCP 客户端-服务器聊天程序
> 实现一个简单的一对一字符界面聊天工具。服务器端监听指定端口，客户端连接后，双方可互相发送文本消息，输入特定指令（如 quit）则断开连接。要求处理基本的连接异常，保证消息传输的可靠性。

2. UDP 广播消息发送与接收
> 编写 UDP 广播程序：一台主机作为发送端，向局域网广播指定端口发送消息（如当前时间、自定义文本）；多台主机作为接收端，绑定该端口接收广播消息并显示。以此理解 UDP 无连接、广播传输的特点。

3. TCP 文件传输工具
> 实现客户端向服务器传输单个文本文件的功能。客户端读取本地文件内容，通过 TCP 套接字分段发送；服务器接收数据并写入新文件，传输完成后校验文件大小是否一致。

4. 多线程 TCP 聊天室
> 基于基础聊天程序升级，服务器支持同时连接多个客户端。当某一客户端发送消息时，服务器将消息转发给其他所有在线客户端，实现群聊功能。需使用多线程处理每个客户端的连接，避免阻塞。

5. 模拟 HTTP 简单请求与响应
> 编写一个迷你 HTTP 服务器，能够处理客户端的 GET 请求。当客户端（可直接用浏览器）访问 `http://服务器IP:端口/文件名` 时，服务器读取本地对应文本文件，返回包含状态码、响应头的 HTTP 格式数据；若文件不存在，返回 404 Not Found。

6. 基于 UDP 的可靠文件传输（模拟停等协议）
> 利用 UDP 实现可靠的文件传输，模拟停等协议的核心逻辑：发送端分段发送文件数据，每发送一个数据段就等待接收端的确认（ACK）消息，超时则重传；接收端收到数据后校验，无误则回复 ACK，否则丢弃该段。

7. 局域网设备扫描工具
> 编写程序扫描指定 IP 段内的主机是否存活，以及开放的常见端口（如 80、443、22）。可通过发送 ICMP 回声请求（模拟 ping 命令）判断主机存活，通过尝试 TCP 连接判断端口开放状态。

8. 简易即时通讯协议设计与实现
> 自定义一套简单的应用层协议（如包含消息类型、发送方 ID、数据长度、数据内容的格式），基于 TCP 实现客户端与服务器的消息传输，支持文字、表情（预设字符组合）等不同类型消息的解析与展示。

9. 其它题目：ping  tracert 抓包分析等  ，也可自拟相关题目
